# Tokenmeter — Claude Code Working Guide

## Project Overview
Tokenmeter is a Windows Electron desktop app that reads local Claude Code JSONL session files, aggregates token usage, estimates costs, and displays a dark terminal-style dashboard. Gemini CLI support exists in the backend but is disabled in the UI (no session files found; re-enable when Gemini CLI local logging is confirmed working).

## Architecture

| File/Folder | Role |
|---|---|
| `main.js` | Electron main: file I/O, IPC handlers, auto-refresh timer, system tray, native notifications |
| `preload.js` | contextBridge — exposes safe `window.tokenmeter` API to renderer |
| `renderer/index.html` | App shell: title bar, overview page, Claude detail page, bottom nav, settings modal, toast |
| `renderer/styles.css` | All styles — pure neutral dark grays, no blue tints, orange (#e8650a) accent |
| `renderer/app.js` | UI logic: routing, chart rendering, heatmap, sparklines, cost alert, idle timer |
| `renderer/animations.js` | PixelEngine (inlined) + all 7 creature presets + cycling controller |
| `src/scanner.js` | Orchestrates Claude + Gemini parsers, builds `combined` totals |
| `src/claude-parser.js` | Parses `.jsonl` files, aggregates tokens/cost/daily/hourly/heatmap/sparklines |
| `src/gemini-parser.js` | Parses Gemini session files (backend ready, UI hidden) |
| `src/pricer.js` | Cost calculation via `pricing.json` pattern matching; `calcCacheSavings()` |
| `pricing.json` | User-editable model pricing table |
| `assets/icon.ico` | App icon — generated by `scripts/gen-icon.js` (do not hand-edit) |
| `scripts/gen-icon.js` | Generates multi-size `assets/icon.ico` + `assets/icon.png` programmatically |
| `scripts/gen-banner.js` | Generates `images/banner.svg` — animated SVG with SMIL creature for README |

## Key Rules
- `contextIsolation: true`, `nodeIntegration: false` — all file I/O in main.js only
- Use `process.env.USERPROFILE` for Windows home path (not `os.homedir()`)
- Claude projects folder uses `--` as path separator in folder names
- All cost estimates labelled as `~$` throughout the UI
- `LOOKBACK_DAYS = 90` in parsers — skip files older than 90 days
- Never use blue tints in UI — palette is pure neutral grays + orange only

## Data Sources
- Claude Code: `%USERPROFILE%\.claude\projects\**\*.jsonl`
- Gemini CLI: `%USERPROFILE%\.gemini\tmp\chats\session-*.jsonl` (backend only, UI removed)
- Pricing config: `pricing.json` (bundled, user-editable)
- App config: `electron-store` → `%APPDATA%\Tokenmeter\config.json`

## Settings (electron-store keys)
```
refreshInterval  — seconds between auto-refresh (default 60)
lookbackDays     — days of history to show (default 14)
claudePath       — override for Claude projects folder
geminiPath       — override for Gemini sessions folder
idleTimeout      — seconds before idle animation triggers (default 60)
dailyCostAlert   — USD threshold for native cost alert (default 0 = off)
```

## Data Flow: claude-parser.js → scanner.js → IPC → renderer
`aggregateClaude(claudeDir)` returns:
- `totalInputTokens`, `totalOutputTokens`, `totalCacheReadTokens`, `totalCacheWriteTokens`
- `totalTokens`, `estimatedCostUSD`, `totalSessions`
- `daily[14]` — 14-day array for charts
- `heatmap[90]` — 90-day array for heatmap grid
- `hourly[24]` — per-hour token totals for peak hours chart
- `cacheSavingsUSD` — accurate per-record savings via `calcCacheSavings(model, cacheReadTokens)`
- `costProjection30d` — 7-day rolling average × 30
- `modelBreakdown` — map of model → { inputTokens, outputTokens, estimatedCostUSD }
- `projectBreakdown[]` — sorted by tokens, each with `sparkline[14]` (14-day token array)
- `recentSessions[10]` — latest sessions by mtime

## Renderer: What renders where
- **Overview page** (`page-overview`): 4 stat chips, Claude CLI row, 14-day daily chart
- **Claude page** (`page-claude`): last active card, insight cards (cache savings + projection), usage stat grid, 14-day chart, 90-day heatmap, peak hours chart, model table, projects table with sparklines, recent sessions
- **Bottom nav**: 2 tabs — Overview, Claude Code
- **Creature**: fixed bottom-left, grounded; cycles IDLE_POOL + EXPRESSIVE_POOL; locks to THINK during refresh
- **Toast**: `#toast` div, bottom-right, auto-dismisses after 3.5s
- **Settings modal**: `#settings-overlay`, toggled via `.visible` class

## IPC Channels
| Channel | Direction | Purpose |
|---|---|---|
| `get-usage-data` | renderer → main | Triggers scan, returns full data object |
| `get-settings` | renderer → main | Returns current settings |
| `save-settings` | renderer → main | Persists settings, restarts refresh timer |
| `window-minimize/maximize/close` | renderer → main | Window controls |
| `open-external` | renderer → main | Opens URL in browser |
| `show-notification` | renderer → main | Fires native OS notification |
| `usage-updated` | main → renderer | Push update after auto-refresh scan |

## Build
```
npm start          # dev mode
npm run build      # NSIS installer + portable exe → dist/
npm run gen-icon   # regenerate assets/icon.ico + assets/icon.png
npm run gen-banner # regenerate images/banner.svg
```
Requires Windows Developer Mode enabled (Settings → System → For developers) for the build step (symlink permissions needed by electron-builder's winCodeSign cache).

Output: `dist/Tokenmeter Setup 1.0.0.exe` (installer) and `dist/Tokenmeter 1.0.0.exe` (portable).

## Project Name Decoding
Folder names under `.claude/projects/` use `--` as separator:
- `C--Users-Dewashish-Lambore-desktop-projects-burnlink` → `burnlink`
- `C--Users-Dewashish-Lambore` → `Home`
- Split on `-desktop-projects-` to extract project name, or use last segment

## Pixel Creature (claudepix)
- 20×20 CSS grid, `BODY=1` cells get color, `EYE=2` and `0` are transparent
- 7 presets: BREATH, BLINK, THINK, DANCE, WINK, SURPRISE, SWAY
- `IDLE_POOL` cycles randomly; every 5th cycle uses `EXPRESSIVE_POOL`
- `setCreatureState('working')` locks to THINK; `'idle'` resumes cycling
- Color: `#CD7F6A` (warm salmon-orange), positioned fixed bottom-left above nav
